## 什么是linux

众所皆知的，linux的核心圆形是1991年由托瓦兹(linus torvalds)写出来的。

但是torvalds为什么可以写出来linux这个操作系统

为什么他要选择386的计算机来开发？

为什么linux的发展可以这么迅速

又为什么linux是免费的。

以及目前为什么由这么多的linux distributions呢

了解这些东西之后，才能够知道为什么linux可以免除专利软件之争

并且了解到linux为什么可以同时在个人计算机与大型主机上面大放异彩。

所以，在实际进入linux的世界之前，就让我们来谈一谈这些有趣的历史故事吧。

### 什么是linux?

linux这个玩意是在计算机上面运作的。

所以，当然得要了解一下计算机这个玩意。

首先，到底有哪些种类的计算机呢？

而linux可以在哪些种类的计算机上面运作的呢？

而linux源自哪里呢？

这些我们都需要来谈一谈先。

### 计算机： 计算的辅助工具

在目前的都市生活当中，您应该很难不接触到计算机这个玩意的吧？

这个计算机当初在开发的时候，是希望可以辅助与简化人们进行大量的运算工作，后来才发展成为一些特殊用途。

无论如何，计算机基本的功能就是：

接受使用者输入命令，经过由CPU的数学与逻辑单元运算处理之后，以产生或储存成有用的信息。

为了达成这个功能，当然计算机就必须有：

输入单元：例如鼠标，键盘，卡片阅读机等等。

中央处理器（CPU）：含有算数逻辑，控制，记忆等单元

输出单元：例如屏幕，打印机等等。

如果您看过计算机的话，那么上面的东西其实就是组成您计算机的很重要组件了。

而且为了连接各个组件，因此有了主机板的。

所以您的主机李曼主要就是包含了主机板以及CPU。

还有各种需要的适配卡。

而屏幕，键盘，鼠标则透过与主机的连接，就构成了一部可以运作的计算机了。

另外，由于计算机仅仅认识0/1，因此计算机主要使用二进制的方式来计算的。

因此，通常计算机的记忆/储存单位都是用byte 或者bits为基本单位的。

他们的单位是这样：

1 bytes = 8 bits

1 kb = 1024 bytes 

1 mb = 1024 kb

1 gb = 1024 mb 

而计算机也因为他的复杂度，而分为各种等级，

例如

超级计算机(supercomputer):

超级计算机是运作速度最快的计算机，

但是他的维护，操作费用也是最高的。

主要是用于需要有高速计算的计划当中的。

例如：国防军事，气象预测，太空科技，用在模拟的领域较多的。

详情也是可以参考的。

国家高速网络与计算机中心http:www.nchc.org.tw/的介绍的。

至于全世界的500大超级计算机，则请参考

http://www.top500.org 

大型计算机(mainframe computer)

大型计算机通常也是具有多数个高速的CPU，功能上虽然是比不上超集计算机的。

但是也可以用来处理大量的资料和复杂的运算。

例如大型企业的主机，全国性的证券交易所等等每天需要处理数百万笔数据的企业机构的。

或者是大型企业的数据库服务器等等。

迷你计算机(minicomputer)

迷你计算机仍然保留有大型计算机同时支持多个使用者的特性。

但是主机可以放置在一般的作业场所，不必像前面的两个大型计算机所需要的特殊的空调场所。

通常用来作为科学研究的，工程分析的与工厂的流程管理的等等。

微型电脑(microcomputer)

又可以称之为个人计算机，也是我们这本书当中主要要探讨的目标的！

体积是很小的，价格是很低的。

但是功能上还是五脏俱全的。

大致上呢，又可以分为桌上型，笔记型等等。

虽然在目前个人计算机的使用是什么广泛的。

但是在1990年以前的时候，个人计算机是比较不受到重视的。

因为他的运算速度在当时的时候是非常差的。

而且当时比较有名的操作系统也没有对个人计算机支持。

所以才会流行不太起来的。

嘿嘿，提到操作系统，下面我们就来谈一谈之前的操作系统的吧。

### 什么是操作系统 

什么是操作系统的(operation system,os)呢？

我们先来想一想，上面的极少的计算器(计算机)是如何工作的呢？

举例来说的话，您计算机屏幕上面显示的讯息，是如何显示出来的呢？

嗯嗯，是由显示卡和屏幕显示出来的。

那么如果你想要看VCD呢？

呵呵，那就是需要

第一，有影音数据的光盘片

第二，能够读取光盘片的光驱。

第三，可以转换影音数据输出到中央处理器的CPU

第四，就是可以显示影响的显示芯片，就是显卡

第五，可以传输声音的音效芯片，就是声卡

第六，可以输出影响的屏幕 

第七，可以发出声音的喇叭

也就是说，所有在工作的东西都是硬件的。

对的，就是硬件在工作的。

那么问题就来了

现在我们知道，计算机所有进行的工作都是计算机硬件帮我们达成的。

但是为什么这些硬件是知道如何播放VCD呢？

这个当然是因为有某种东西在正确的控制硬件的工作了。

那个东西就是操作系统了。

操作系统可以管理整个计算机的硬件。

他可以控制cpu进行正确的运算

他可以分辨硬盘里头的数据并且进行读取的

他还必须要能够认识所有的适配卡

这样，才能够将所有的硬件统统正确的使用上嘎。

所以，如果没有这个操作系统，那么您的计算机就是等于是一堆废铁而已的。

虽然操作系统可以完整的掌控所有的硬件资源

但是，对于，使用者来说，这个还是不够的。

因为操作系统虽然可以掌控所有的硬件。

但是，如果使用者无法与操作系统进行沟通，那么这个操作系统就没有什么用处了。

简单的来说，以上面的VCD为例，虽然操作系统可以控制硬件播放VCD。

但是，如果使用这么诶有办法控制合适要播出VCD的话，那么我们到底要怎么看VCD呢？

对吧。

所以说，一个比较完整的操作系统应该是要包含两个东西的，

一个是核心和其提供的接口工具。

一个是利用核心提供的接口工具所开发出来的软件。

我们以为大家常常使用的windows计算机来做一个简单的说明好了。

大家应该都是使用 过windows计算机里面的档案总管的吧。

当你开启档案总管的时候，他就会显示你硬盘当中的数据的。

这个【显示硬盘里面的数据，就是核心帮你做的】。

但是，你要核心区显示硬盘的哪一个目录下面的数据，则是由档案总管这个工具帮你达成的，这样可以理解的么？

那么核心有没有做不到的事情呢？

当然是有的。

举例来说的话，如果您曾经自己安装过比较新的显卡在您的个人计算机上面。

那么应该常常会发生windows计算机机告诉你，找不到合适的驱动程序来显示的问题的吧

也就是说，即时你有最新的显示卡安装在您的个人计算机上面，而且来说，也有播放VCD的程序

但是因为核心是没有办法操控这个最新的显示卡的。所以，也就无法正常显示你的VCD的。

没有错误的，您的整个硬件都是由核心来管理的。而且如果核心不认识你的硬件的话。

那么你就无法使用这个硬件设备的。

例如上面提及的最新的显卡。

tips

在定义上，只要是能够让计算机硬件正确无误的运作的，那么就算是操作系统了。

所以说，操作系统实际上就是核心和核心提供的接口工具的。

不过，就如同上面讲解的那个样子的，因为最重要的核心如果是缺乏了跟使用者沟通的亲和的借口。

那么你的使用体验也是非常差的吧。

所以来说，在目前来讲的话，一般我们提及的操作系统都会包含一些核心和相关的使用者的应用软件额。

核心就是kernel。

他是一个操作系统的最底层的东西，由他来掌管整个硬件资源的工作状态的。

而且每个操作系统都是有自己的核心的。

素偶已说，当有新的硬件加入到你的操作系统当中的时候

你的kernel并没有支援他的时候，呵呵，这个新的硬件即时肯定无法工作的。

因为控制他的kernel并不认识他的呀

这样了解了么

先有一个概略性质的了解，后面我们提到了核心编译的时候会更加详细的谈到他的。

一般来说，kernel为了达成使用者所需要的正确运算即诶过，他必须要管理的事项有下面的几个部分的。

系统呼叫借口: system call interface 

为了方便程序开发者可以轻易的透过与kernel的沟通，将硬件的资源进一步的利用。

于是需要有这个建议的接口来方便程序开发者。

进程管理(process  control)

你应该是听过什么叫做多任务环境的吧。

一个计算机可能同时间是有多个工作跑到CPU当中等待运算处理的。

kernel这个时候必须要能够控制这些工作的，让CPU的资源有效分配才是可以的。

内存管理 memory management 

控制整个系统的内存管理

如果你的内存不足的话，kernel最好还是能够提供虚拟内存的功能的。

文件系统管理 file system management 

文件系统的管理，比如说是数据的输入和输出等等的工作啦。

还有不同的文件格式的支持啦等等的。

如果你的kernel不认识你的文件系统，那么你就没有办法使用这个文件的

比如说windows98就是不认识NTFS的文件格式的硬盘的。

硬件驱动 device drivers 

这个就是上面提及的，硬件的管理是kernel的主要工作之一了。

当然了，加载硬件驱动程序就是kernel需要做的事情叻

好在目前都有所谓的可加载模块的功能。

可以将驱动程序编辑成为模块，这样就不需要重新的编译kernel了。

这个也在后续的kernel编译当中会提及的。

好了，这里我们讲解了kernel必须管理的几个部分

进程，内存，文件系统，驱动，系统接口。

所以啦，所有的硬件的资源都是kernel来管理的。

至于我们要达成一些工作的时候，除了由整个核心本身提供的功能(例如上面提到的文件系统)之外。

还可以由其他的应用软件来达成的。

举个例子来说，你要看VCD影片是吧。

那么除了windows提供的媒体播放程序之外，你可以自己安装VCD播放程序播放VCD了。

这个播放程序就是应用软件啦。

这个应用软件是可以帮你去控制核心来进行工作的，就是放映影片了。

因此，我们可以这样说的，kernel就是控制整个硬件的这个东西，就是操作系统的最底层的东西，

然而想要整个操作系统更加完备的话，那么还是需要含有相当丰富的kernel提供的工具，以及kernel相关的应用软件来支持的。
